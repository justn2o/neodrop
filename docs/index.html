NeoDrop

Send files and folders straight to another computer — with a short code.

No server. No account. End-to-end encrypted.

View on GitHub Download

Why NeoDrop

It goes from one machine to the other — nothing is uploaded to the cloud.

📁

Files & folders

One file, many files, or a whole folder — the structure is rebuilt exactly on the other side.

🔑

Simple pairing

You get a short code like TIGER-7342 (and a QR code to scan). The other person just types it in.

🔒

End-to-end encrypted

Direct, encrypted connection between the two computers. Every file is SHA-256 verified on arrival.

⚡

Fast on a LAN

Two computers on the same network find each other in milliseconds.

♻

Resume after a drop

If the connection cuts out, restart with the same code and it picks up where it left off.

⚙

You stay in control

The receiver sees what’s coming, picks where to save it, and nothing is written until they accept.

How it works

Three steps, both ways.

Send — drop your files or a folder onto the app. You get a short pairing code (and a QR code).

Share the code — the other person, anywhere, enters it. If it’s on their clipboard it’s filled in for them.

Receive — they confirm, pick a folder, and the transfer runs with progress on both sides, then an integrity check.

Built to be private

Security is the default, not an option.

  • No server — peers find each other through a public distributed network and connect directly.
  • End-to-end encryption (Noise protocol) on every connection, including on the local network.
  • The code never travels in clear — a mutual challenge-response proves both sides know it, bound to the connection to defeat man-in-the-middle.
  • Optional passphrase and stronger codes for sensitive transfers.
  • SHA-256 integrity check — what arrives is exactly what was sent, or it’s rejected.
