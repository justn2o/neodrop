From send to verify, in four steps

No setup, no sign-up. The two peers find each other through the public Hyperswarm DHT, or directly on the local network, and talk with no relay.

01

Drop

Drag files or a whole folder. A short code is generated, along with a QR code.

02

Share the code

Pass the code to the other person. If it's already on their clipboard, it fills in.

03

Confirm

The recipient confirms and picks the destination folder. Nothing is written before they agree.

Verify

Progress on both sides, then an integrity check.

The shortest path between two disks

NeoDrop looks for the two machines in parallel: over the internet via the DHT, and on the local network via mDNS. The first to answer wins.

Two paths, one encryption

Whether the connection goes over the internet or the local network, the same Noise protocol encrypts the socket. End-to-end confidentiality doesn't depend on the path.

Hyperswarm DHT. Global discovery, with UDP hole punching to cross NATs. An encrypted relay takes over if hole punching fails.
Local mDNS. On the same network, peers find each other in milliseconds, and it works even without internet.

Files and folders

A folder's whole tree is recreated exactly on the recipient side, with a preview for images.

If the connection drops, restarting with the same code resumes where it left off instead of resending everything.

The code never travels in clear

Confidentiality is in the structure of the protocol, not a checkbox. Here's what protects a transfer, primitive by primitive.

noise · hyperswarm

End-to-end encryption

Sockets are encrypted natively by the Noise protocol, local network included. No middleman reads the content.

scrypt → hkdf

Unguessable DHT topic

The topic is derived from the code with scrypt, costly by design, then HKDF. Watching the DHT isn't enough to brute-force it.

hmac-sha256

Mutual challenge-response

Each peer proves it knows the code via an HMAC over random nonces, with reflection protection. The code itself never travels.

3 tries · 15 min

Single-use code

Three failed attempts invalidate the code on the sender side, and it expires anyway after fifteen minutes with no connection.

passphrase

Secret on top of the code

Folded into the topic and the auth key, a passphrase makes pairing impossible even if the code is intercepted.

sandbox · contextIsolation

Isolated interface

The Electron renderer is sandboxed; all network and file logic lives in the main process. Received names are sanitized before writing.

The same thing, with no UI

Scriptable and automatable, with identical encryption guarantees. Installed globally, the command is neodrop.

--strength high|max --pass PHRASE --limit MB/s --out DIR --yes
neodrop
# send a folder, stronger code and passphrase
$ neodrop send photos/ --strength high --pass "holiday"
→ pairing code FALCON-2914
→ waiting for the recipient
# on the other machine
$ neodrop receive FALCON-2914 --out ./received --yes
✓ 248 files · 1.4 GB · SHA-256 verified
$
